Computational Journalism Publishers Workbench 2.0.0 Released

Feb 18th, 2013 | Comments

Quick Start

I’ve just pushed release 2.0.0 of the Computational Journalism Publishers Workbench to Github. I’ve shuffled a few things around, removed some tools I wasn’t using and added some new ones. If there’s anything missing from previous releases that you want back, either open an issue on Github or leave a comment here.

What’s New?

  1. Bare metal support: I’m now testing on 64-bit machines with both Linux Mint 14 and Fedora 18. You’ll need an Intel or AMD 64-bit machine with at least 4 GB of RAM. Both Fedora 18 and Linux Mint 14 have excellent installation documentation.
  2. Virtual hosting on bare metal: I’ve included scripts to install the Virtual Machine Manager and the kernel virtual machine hypervisor. See the Virtual Hosting README.md.
  3. Massive refactoring: I removed a number of packages that I wasn’t using, and added many more optional end-user tools from the Linux repositories. See the README.md files in the directories for the major components.
  4. RKWard: it turns out that the RKWard R GUI/IDE is available for both Fedora 18 and Linux Mint 14. This isn’t quite as powerful as RStudio, but it’s packaged by the Linux teams and has a fair amount of help available for beginners.
  5. Suggested readings: I’ve added Amazon affiliate links for some of the better books for beginners on the technologies available. I own most of them already and the rest are on my wish list.

Road Map

  1. Now that this release is done, I’ll be revising the CompJournoStick code to draw from this repository in constructing media. That way, the bill of materials for the stick will be the same as the Fedora version of this workbench.
  2. I’m backing the For Journalism project on Kickstarter. I’m tracking their materials and will be adding software to this project and CompJournoStick to run all of their code.
  3. I’ve looked at openSUSE 12.3 and there are some critical tools missing from their repositories. So I will not be supporting openSUSE going forward.
  4. I’m planning to support Ubuntu 13.04 and Linux Mint 15. I don’t have enough disk space on my test machines to test Ubuntu on bare metal, however.
  5. The next release will have Redis 2.6 back in. I removed it because the Linux repositories are still shipping 2.4, and I wanted to install it from Linux rather than build it from source.
  6. The next release will have Shiny server and Node.js. It may also have Meteor, which seems to be the up-and-coming wrapper for Node.js.
