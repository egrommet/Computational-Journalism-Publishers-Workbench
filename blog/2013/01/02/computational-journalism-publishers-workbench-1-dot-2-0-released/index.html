Computational Journalism Publishers Workbench 1.2.0 Released

Jan 2nd, 2013 | Comments

Quick Start

  • Quick Start - Generic Linux on VMware Player
  • Quick Start - Fedora 17 / 18 Beta on VirtualBox
  • Quick Start - Linux Mint 14 on VirtualBox

What’s New?

  1. There is now alpha-level support for hosting with Windows 8 Pro’s ‘Client Hyper-V’ virtualizer. I’ve only tested it with a Fedora 18 64-bit guest, but the performance on the Redis benchmark is good enough that I’m going to support it. There are some things you’ll need to know:
    • You’ll need two network adapters if you want to run the host Windows machine and the guest workbench at the same time. The Hyper-V ‘Virtual Switch’ connects to one of the network adapters and locks out the host when it does.
    • You can’t run either VMware or VirtualBox in the machine if you have Hyper-V enabled.
    • Turning Hyper-V on or off requires a reboot.
  2. Fedora, Linux Mint, openSUSE and Ubuntu are now supported as guests with VirtManager / qemu-kvm. The host version I’m using is what’s shipped with Fedora 18, but it should work with other Linux distributions. Mageia 2 does not appear to work as a guest in this system; if you want to use Mageia, use either VirtualBox or VMware Player / Workstation.
  3. I’ve tuned the Redis benchmark in the ‘Redis-Source’ directory. And on Fedora 18 or Linux Mint 14, it will use ‘operf’ to capture a profile of the Redis process being benchmarked. This also works with in a guest system running on a VirtManager / qemu-kvm host, provided the Processor configuration uses the host processor topology. See the sub-directories in https://github.com/znmeb/Computational-Journalism-Publishers-Workbench/tree/master/Redis-Source for benchmark results with Fedora 18 on bare metal and guest systems.
  4. I’ve updated PDQ to the 6.0.1 release.

Road Map

  1. Fedora 18 is scheduled for release on January 8, 2013. I’ve been running Fedora 18 as my main release since late alpha stage and it’s solid. The only major hitch is that the installer user interface is brand-new and will probably confuse you if you’re used to the old one. For installation in a guest on blank hard drives, this isn’t a problem - you just accept the defaults and go. But if you’re dual-booting it on a Windows machine you’ll want to hunt down some documentation before doing an install. In any event, I will stop supporting Fedora 17 once 18 is released.
  2. The R Twitter API package is now in beta test with the Twitter 1.1 API. I’ll be testing it in the near future and will push out a release once I have some demo code for it.
  3. I haven’t started serious work with Mageia 3 yet. I’ve run a few preliminary tests with the alpha media but I ran into too many issues and abandoned it. Mageia 2 is fine for now.
  4. Longer term (around April 2013) R 3.0 is being released. At that point or shortly before, I will migrate a fair amount of software that’s now being installed from source or from upstream binaries into the dependency structures of the distributions. R and TeXLive in particular will be moved into the ‘platform’ part of the workbench. But just about everything that’s already packaged in both Fedora and Linux Mint will be taken from the repositories, and I will be dropping support for openSUSE and Mageia if these packages aren’t maintained there.
