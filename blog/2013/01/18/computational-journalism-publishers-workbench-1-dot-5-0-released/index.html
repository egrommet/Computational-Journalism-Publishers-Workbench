Computational Journalism Publishers Workbench 1.5.0 Released

Jan 18th, 2013 | Comments

Quick Start

  • Quick Start - Fedora 18 or Linux Mint 14 on VMware Player
  • Quick Start - Fedora 18
  • Quick Start - Linux Mint 14

What’s New?

  1. The preferred hosting for virtual machines is now VMware Player or VMware Workstation. I haven’t done any comprehensive performance testing, but for the Redis benchmarks in https://github.com/znmeb/Computational-Journalism-Publishers-Workbench/tree/master/Redis-Source, VMware was much faster than VirtualBox.
  2. I’ve dropped support for Ubuntu, Mageia, and openSUSE:
    • The testing time was too much for me.
    • Ubuntu 12.10 has some issues with the default Unity desktop in virtual machines.
    • Mageia 2 has issues with the display as a Virtual Machine Manager / GNOME Boxes guest.
    • openSUSE 12.2 is missing some packages that I added for this release.
  3. I’ve added the Padre Perl IDE.
  4. PostgreSQL is now installed as part of the platform.
  5. I’ve added PostGIS to the ‘Spatial’ task view.
  6. I’ve added the Python Natural Language Processing Toolkit (NLTK) to the ‘Natural Language Processing’ task view.
  7. I’ve added R utilities for dealing with semantic web RDF data to the ‘Natural Language Processing’ task view.
  8. I’ve added a new “pseudo-task view” for social network / graph analysis. In addition to R packages, this option contains the Python NetworkX package.

Road Map

  1. With the release of Fedora 18, I consider this project “finished”. I’ll continue to support Linux Mint 14 and fix bugs, but I’m focusing on Fedora 18 and 19 for the remainder of 2013. As I noted above, I was spending a lot of time testing on openSUSE, Ubuntu and Mageia and there wasn’t any real payoff. I will be looking at openSUSE 12.3, which is now in beta, but unless some critical packages show up in it, I won’t be pursuing it further.
  2. The Perl Net::Twitter API package is now in beta test with the Twitter 1.1 API. I’ll be testing it in the near future and will push out a release once I have some demo code for it.
  3. I am going ahead with the plan for a new project based on Fedora 18. It will be a Fedora Remix and will contain most of the software in release 1.5.0. I’m pretty sure I’ll be dropping Maqetta and Google Refine, and I’m pretty sure I’ll be adding Node.js and Django, but otherwise it will look a lot like this release.
